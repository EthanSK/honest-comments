Three steps to honest feedback

You bring the agent and a free key. We bring the operating manual. Nothing to install beyond a CLI you might already have.

  1. 1

    Get Codex CLI or Claude Code

    honest-comments runs inside an AI coding agent you control. Pick whichever you already pay for — it orchestrates the whole flow locally and does the analysis using the subscription you already have.

    OpenAI Codex CLI developers.openai.com → Anthropic Claude Code anthropic.com →
  2. 2

    Get a free YouTube API key

    A plain Google Cloud API key (public data only — no OAuth, no consent screen). It reads public comments on public videos. The key lives in your env/.env and is used only to call Google's YouTube API — it never goes to honest-comments. The agent walks you through it the first time, or you can follow the README.

    • Create a project in the Google Cloud console
    • Enable YouTube Data API v3
    • Create an API key, then set it as the YOUTUBE_API_KEY environment variable or drop it in a .env file (recommended — keeps it out of chat logs)
    • Or, if you prefer, just paste it to your agent — it's your key; the script uses it to call Google's YouTube API and it never goes to honest-comments
    Full key walkthrough in the README →
  3. 3

    Paste this prompt

    Drop this into Codex CLI or Claude Code. It fetches the operating manual and takes it from there — greeting you, helping with the key, fetching comments, and delivering the ranked digest.

    Starter prompt 
    You are running "honest-comments". Your VERY FIRST action: clone the repo and
work from inside it (the script and prompts live there, not in this folder):

  git clone https://github.com/EthanSK/honest-comments && cd honest-comments

If git is unavailable, download and unzip the repo instead, then cd into it:

  curl -L -o honest-comments.zip https://github.com/EthanSK/honest-comments/archive/refs/heads/main.zip
  unzip honest-comments.zip && cd honest-comments-main

Then read README.md from this local checkout in full and follow it as your
instructions. Run every later command from inside this cloned directory.

I'm a YouTube creator. Your job: dig through my comments and surface the
criticism that's actually worth acting on — ranked, with concrete fixes. Skip
the praise, filter out the trolls, and don't soften the useful-but-harsh stuff.

Once you've read the manual, run the whole flow conversationally:
1. Greet me and explain what you're about to do.
2. Help me get a free YouTube Data API key if I don't already have one.
   Prefer setting it as an env var or in a .env file so it stays out of chat

   honest-comments.
3. Ask whether I want my whole channel or specific videos, and confirm scope

Be honest, not nice. Begin now by greeting me.

    That's the whole onboarding — one fetch, one instruction. Updates ship via the README, so you never re-paste.

The model

Why we can promise we see nothing

There's no "we" in the data path — honest-comments is just a GitHub repo and this page, with no servers. Your key talks to Google's YouTube API to fetch comments, and your own AI (the Codex/Claude plan you already pay for) classifies them — exactly like any prompt you send it. Nothing routes through us.

BYO-agent

The analysis runs in your Codex CLI or Claude Code. We never run a model on your behalf — there's nothing for us to log.

No backend

No servers, no database, no API key of ours. Your fetched comments are saved to a local out/ folder; your YouTube key lives in your env/.env and is only ever sent to Google's YouTube API. Nothing routes through honest-comments.

Your subscription

It uses the AI plan you already pay for. No new account, no per-use billing from us, no surprise costs.

Open source

Every line is on GitHub under MIT. The fetch script is stdlib-only Python — read it, fork it, run it offline if you like.

The payoff

What lands in your terminal

Not sentiment. Not a word cloud. A ranked list of the things real people are asking you to fix — most-agreed-upon first, each paired with a concrete fix.

honest-comments · @YourChannel · newest 25 videos

Of 4,210 comments: ~3,100 praise, ~260 trolls filtered out. These 6 themes are the signal worth acting on.

#1

Audio is too quiet / voice gets buried

High

11 people raised this · +340 likes

"Great info but I had to max my volume and your voice still clips."
"bg music way louder than you the whole video"

Fix Raise voice gain and add a compressor; aim for ~-14 LUFS integrated. Duck music ~12 dB under your voice.

#2

Intro is too long

Medium

7 people raised this · +96 likes

"90 seconds of intro before the actual content, please get to the point"

Fix Cut the cold-open to under 15s; put the payoff promise in the first 10 seconds.

#3

Code font too small on mobile

Medium

5 people raised this · +58 likes

"Hard to read the code, font's too small on my phone"

Fix Bump editor font to ~18–20px for screen recording; zoom into the active region instead of showing the whole file.
