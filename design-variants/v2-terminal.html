setup --takes 3-minutes

Three steps to honest feedback

You bring the agent and a free key. honest-comments brings the operating manual. Nothing to install beyond a CLI you might already have.

  1. step-1 · get-an-agent
    01

    Get Codex CLI or Claude Code

    honest-comments runs inside an AI coding agent you control. Pick whichever you already pay for — it orchestrates the whole flow locally and does the analysis on the subscription you already have.

    OpenAI Codex CLI developers.openai.com → Anthropic Claude Code anthropic.com →
  2. step-2 · youtube-api-key
    02

    Get a free YouTube API key

    A plain Google Cloud API key (public data only — no OAuth, no consent screen). It reads public comments on public videos. The key lives in your env / .env and is used only to call Google's YouTube API — it never goes to honest-comments. The agent walks you through it the first time.

    • Create a project in the Google Cloud console
    • Enable YouTube Data API v3
    • Create an API key, then set it as the YOUTUBE_API_KEY env var or drop it in a .env file (recommended — keeps it out of chat logs)
    • Or paste it to your agent — it's your key; the script calls Google's YouTube API with it and it never goes to honest-comments
    open README --section key-walkthrough ↗
  3. step-3 · paste-the-prompt
    03

    Paste this prompt

    Drop this into Codex CLI or Claude Code. It clones the repo, fetches the operating manual, and takes it from there — greeting you, helping with the key, fetching comments, and delivering the ranked digest.

    starter.txt — paste this whole block 
    You are running "honest-comments". Your VERY FIRST action: clone the repo and
work from inside it (the script and prompts live there, not in this folder):

  git clone https://github.com/EthanSK/honest-comments && cd honest-comments

If git is unavailable, download and unzip the repo instead, then cd into it:

  curl -L -o honest-comments.zip https://github.com/EthanSK/honest-comments/archive/refs/heads/main.zip
  unzip honest-comments.zip && cd honest-comments-main

Then read README.md from this local checkout in full and follow it as your
instructions. Run every later command from inside this cloned directory.

I'm a YouTube creator. Your job: dig through my comments and surface the
criticism that's actually worth acting on — ranked, with concrete fixes. Skip
the praise, filter out the trolls, and don't soften the useful-but-harsh stuff.

Once you've read the manual, run the whole flow conversationally:
1. Greet me and explain what you're about to do.
2. Help me get a free YouTube Data API key if I don't already have one.
   Prefer setting it as an env var or in a .env file so it stays out of chat
   logs; the script uses it to call Google's YouTube API. It never goes to
   honest-comments.
3. Ask whether I want my whole channel or specific videos, and confirm scope
   (run scripts/fetch_comments.py with --dry-run to show me the quota estimate)
   before fetching anything.
4. Run scripts/fetch_comments.py to pull my comments.
5. Apply prompts/analyze.md to classify, dedupe, cluster, and rank the
   constructive criticism — ignoring praise and trolls.
6. Deliver the ranked harsh-but-useful takes, then offer to go deeper.

Be honest, not nice. Begin now by greeting me.

    That's the whole onboarding — one paste, one instruction. Updates ship via the README, so you never re-paste.

trace --where-does-my-data-go

Where your data actually goes

We won't pretend "nothing leaves your machine" — that wouldn't be true, and you'd be right not to trust it. Here's the honest map. The one thing that's certain: nothing routes through honest-comments.

→ Google's YouTube API

Your key + fetch requests

Your YOUTUBE_API_KEY stays in your env / .env and is used locally to call Google's YouTube API — the only place the key goes, because that's how public comments get fetched.

→ Your own AI provider

Comment text for analysis

The comments are classified by your Codex / Claude subscription — exactly like any other prompt you send it. It leaves to your model provider, the same as any agent use.

→ nowhere

honest-comments receives nothing

No backend, no account, no servers. honest-comments is a GitHub repo and this page. There is no "us" anywhere in the data path to log, store, or sell anything.

Your agent → Your YouTube key → Comments on your disk → Ranked digest

It uses the AI plan you already pay for. No new account, no per-use billing from us, no surprise costs. Every line is on GitHub under MIT — the fetch script is stdlib-only Python. Read it, fork it, run it offline if you like.

Audio is too quiet / voice gets buried

High

Medium
